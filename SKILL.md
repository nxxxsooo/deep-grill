---
name: deep-grill
description: Stress-test a plan or design by grilling it yourself before asking the user. Use when the user says 'deep grill', or wants a plan stress-tested autonomously. For interactive one-question-at-a-time grilling, use 'grilling' instead.
---

Interrogate this plan relentlessly — but answer your own questions before asking the user. Walk every branch of the decision tree, resolving dependencies one by one: investigate each question (read docs, explore the code, dispatch subagents), name the best answer, then argue against it before accepting it.

Only genuinely subjective calls go to the user. Surface them in one consolidated batch, each with your recommended answer.

Do not enact the plan until the user confirms we have reached a shared understanding.
